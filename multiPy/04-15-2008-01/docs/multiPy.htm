pyProxy 1.0

 

If you are like me and you have many Python versions all installed at the same time on your local drive C:\ for Windows XP Pro (works for any Windows versions) then you will want to be able to switch between Python versions at a moment’s notice.

 

pyProxy makes switching between Python versions fast and easy.

 

Consider the following Python versions all installed on drive C:\.

 

C:\ Python23 ß Python 2.3.5

 

C:\ Python24 ß Python 2.4

 

C:\ Python25 ß Python 2.5.2

 

You see, I have some legacy code written against Python 2.3 running on a Red Hat Linux box along with some newer code running on Windows against Python 2.5.2 and I wanted to be able to play with any of this code whenever I wish just by pressing the right button.

 

So let’s run the Python Interpreter for each version.

The following command runs Python 2.3.5 and slams me into the Python Interpreter in immediate mode.

 

pyProxy 23 <enter>

 

That’s all there is to it !  Nice and easy.

 

Same thing works for the other two Python versions using the following commands:

 

pyProxy 24 <enter>

 

pyProxy 25 <enter>

 

You can exit Python by pressing the Ctrl-Z keys and bingo pyProxy removes whatever Environment settings were required to make the selected Python Interpreter run.

What about the Wing IDE ?

 

Okay, that’s all fine and well when all you wanted was to run the Python Interpreter in immediate mode. 

 

But what about the Wing IDE ?

 

Wing IDE is a great tool for developing and working with Python code and it has a built-in Shell that acts just like the Python Interpreter in immediate mode.

 

So what if you wanted to run Wing IDE for Python 2.3.5 ?

 

Just issue the following command and you be in like flint (so to speak):

 

pyProxy 23 1 <enter>

 

The second argument, “1”, has been coded to automatically run the Wing IDE after the proper Environment vars have been set for Python 2.3.5.

 

You can add whatever additional tools or programs as may be desired using the Windows Batch Script found at the end of this article.

 

What about the Windows Explorer Command Prompt Registry Hack ?  (Windows XP Only)

 

And now after you’ve gotten everything else mentioned in this article to work we come to the icing on the cake.

 

The ability to launch a DOS Command Prompt from the Windows Explorer context menu (right click on a folder name) tailored for the version of Python YOU want to use.

 

Merge all these Registry hacks into your Registry and you too will be able to launch a Python DOS Command Shell tailored to the specific version of Python you need for whatever task for which you need to use Python.

 

The Registry Hack that launches a standard DOS Command Prompt: (Windows XP Only)

 

Windows Registry Editor Version 5.00

 

[HKEY_CLASSES_ROOT\Folder\shell\CmdPrompt]

@="Command Prompt"

 

[HKEY_CLASSES_ROOT\Folder\shell\CmdPrompt\Command]

@="c:\\windows\\system32\\cmd.exe /k \"pushd %L && title Command Prompt\""

 

 

The Registry Hack that launches a DOS Command Prompt for Python 2.3.5: (Windows XP Only)

 

Windows Registry Editor Version 5.00

 

[HKEY_CLASSES_ROOT\Folder\shell\CmdPrompt23]

@="Python 2.3.5 Command Prompt"

 

[HKEY_CLASSES_ROOT\Folder\shell\CmdPrompt23\Command]

@="Z:\\_utils\\pyProxy.cmd 23 2"

What about the Windows Explorer Command Prompt Registry Hack ?  (Windows Vista Only)

 

I have already given you the Windows XP Registry hacks but what about Windows Vista ?

 

This is the Registry hack for Windows Vista that pretty much does the same thing as is mentioned above for Windows XP.  I leave it up to the reader to adapt the previously mentioned Windows XP Registry hacks to work with Windows Vista based on the sample below.

 

Windows Registry Editor Version 5.00

 

[HKEY_CLASSES_ROOT\Directory\shell\runas]

@="Administrator Command Prompt here"

"NoWorkingDirectory"=""

 

[HKEY_CLASSES_ROOT\Directory\shell\runas\command]

@="cmd.exe /k \"pushd %L && title Command Prompt\""

 

[HKEY_CLASSES_ROOT\Drive\shell\runas]

@="Administrator Command Prompt here"

"NoWorkingDirectory"=""

 

[HKEY_CLASSES_ROOT\Drive\shell\runas\command]

@="cmd.exe /k \"pushd %L && title Command Prompt\""

 

 

And now for the code (pyProxy.cmd):

 

@echo off

REM %1 is the version number of the Python we wish to execute.

REM %2 is the program we wish to launch - full path to the executable...

 

REM %2 == 1 for "C:\Program Files\Wing IDE 3.0\bin\wing.exe"

REM %2 == 2 for %ComSpec%

 

SET PYPROXY_1="C:\Program Files\Wing IDE 3.0\bin\wing.exe"

SET PYPROXY_2=%ComSpec% "/k"

 

REM pyProxy 23 1 <-- Runs Wing IDE for Python 2.3

REM pyProxy 24 1 <-- Runs Wing IDE for Python 2.4

REM pyProxy 25 1 <-- Runs Wing IDE for Python 2.5

 

if %1. == 23. goto run23

if %1. == 24. goto run24

if %1. == 25. goto run25

goto runError

 

:run23

goto validVersion

 

:run24

goto validVersion

 

:run25

goto validVersion

 

:validVersion

set old_path = %Path%

set Path=c:\ruby\bin;c:\python%1;c:\python%1\scripts;%systemroot%\system32;%systemroot%;%systemroot%\system32\wbem;c:\program files\hummingbird\connectivity\9.00\accessories\;c:\program files\quicktime\qtsystem\;c:\progra~1\xray;c:\program files\mysql\mysql server 5.0\bin

 

set old_PYTHONPATH=%PYTHONPATH%

set PYTHONPATH=C:\Python%1

 

if %2. == . "%PYTHONPATH%\python.exe"

if %2. == 1. %PYPROXY_1%

if %2. == 2. %PYPROXY_2%

 

set path=%old_path%

set PYTHONPATH=%old_PYTHONPATH%

 

goto exit

 

:runError

echo ERROR - Cannot execute the version "%1".

goto exit

 

:exit

echo Run done.
